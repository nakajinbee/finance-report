## 2. 動作確認

### 実施した確認方法

ブラウザ操作の自動化ツールが本環境になく、Playwright等でのクリック操作による確認は**実施できていない**。
代わりに以下の方法で検証した：

- バックエンド：`curl`で全APIエンドポイントを実データ（リクルートHD、EDINETコード`E07801`）に対して実行し、
  レスポンス形が`frontend/src/api/client.ts`の型定義と一致することを確認
- フロントエンド：`tsc -b --noEmit`・`oxlint`が通ることを確認（型エラー・lintエラーなし）
- ダウンロード→一覧→詳細→CF→facts の一連のAPI呼び出しを実際のダウンロードジョブで通しで確認

- [x] 証券コードで検索して結果が表示されるか（同上）
- [x] 企業を選択して財務指標が一覧表示されるか（`GET .../financials`のレスポンスで確認）
- [ ] 棒グラフが正しく表示されるか　→ **ブラウザでの目視確認は未実施**（データ・型は検証済み）
- [ ] 指標の追加・削除ができるか　→ **ブラウザでの目視確認は未実施**（実装はサイクル1から変更なし）
- [x] キャッシュフローのグラフ・表が正しいデータで表示されるか（APIレスポンス値で確認）

- [x] 検索で0件の場合にメッセージが表示されるか（`CompanyFactsPage`/`CompanyListPage`のコードレビューで確認、実データでは0件ケースは未再現）
- [ ] EDINET API がエラーを返した場合にエラーメッセージが表示されるか　→ **未確認**（意図的にAPIキーを不正化する等の障害注入テストは未実施）
- [x] 対象指標のXBRLデータが存在しない場合に「データなし」と表示されるか（US GAAP企業でoperating_profit/total_liabilitiesが欠落するケースを前サイクルで実データ確認済み、今回のCFグラフ・表でも同じ`formatYenForDisplay(null)`ロジックを再利用）
- [ ] ネットワーク切断時に適切なエラーが表示されるか　→ **未確認**

意図的な障害注入（EDINET APIエラー、ネットワーク切断）による確認は本環境の制約により未実施。
実機ブラウザでの最終確認はユーザー側での実施を推奨する。

## 発見した不具合と対応（開発中に修正済み）

| # | 内容 | 対応 |
|---|---|---|
| 1 | 半期報告書CSVのファイル名プレフィックスが制度移行期のみ`q2r`になり、想定の`ssr`と異なっていた | 両方のプレフィックスを許容するよう修正 |
| 2 | EDINET書類取得APIの`periodEnd`が半期報告書では半期末ではなく事業年度末を返し、`facts.period_end`が誤って保存されていた | 自前算出した期間終了日を保存するよう修正 |
| 3 | 半期報告書義務化（2024年4月1日以後開始事業年度）前の年度も毎回探索し、確実に失敗するログ行が並んでいた | 義務化前の年度は探索自体をスキップするよう修正 |
| 4 | ダウンロード画面に、完了前は他画面へ遷移する手段がなかった（「全期間」指定で数分かかるケースで特に問題） | 常時表示の「企業一覧へ」リンクを追加 |
