# サイクル14 要件定義 セルフレビュー

[self_review_rule.md](self_review_rule.md)に沿って確認した。

## 抜け漏れチェック

- **状態の網羅**：企業一覧画面の状態を以下の4つに整理した。既存の「ローディング」
  「エラー」「保存済み企業が0件（データ未取得）」はサイクル14でも変更なし。
  変更が入るのは絞り込み条件の有無による表示切り替えのみ。
  1. 初期状態（キーワード未入力・業種未選択）→ 案内メッセージのみ、カード非表示
  2. 絞り込み条件あり・該当0件 → 「見つかりませんでした」メッセージ（既存の挙動を流用）
  3. 絞り込み条件あり・該当1件以上 → カード一覧表示
  4. 保存済み企業自体が0件（データ未取得）→ 既存の「データがありません」画面のまま
     （こちらは絞り込み条件と無関係に先に判定する）
- **AND条件の明確化**：キーワード検索と業種絞り込みは併用可能（FR-63）とし、
  「絞り込み条件がある」＝キーワードが空でない、または業種が「すべて」以外、の
  いずれかが真であること、とFR-62に定義した
- **件数表示との整合**：「N件中M件表示」の表示は、初期状態（未絞り込み）では
  そもそもカード一覧を表示しないため、件数表示も初期状態では出さない
  （絞り込み条件がある場合のみ表示）よう設計に反映する

## 実現可能性チェック

- `GET /api/companies`は既に全社分（企業名・証券コード・業種・会計基準）を
  返しており、業種絞り込み・ソートに必要なデータは揃っている。バックエンド・DBの
  変更は不要（スコープ外に明記済み）
- 業種の選択肢は取得済み企業データから`sector`の重複除去で作れるため、
  新規APIは不要

## スコープの明確化

- バックエンド変更なし、URL連携なし、業種区分自体の見直しなしをスコープ外として
  明記した。ユーザーからの依頼は「初期非表示化・業種絞り込み・ソート順」の3点のみで、
  それ以外への拡張は行わない（[feedback_confirm_feature_before_adding]相当のルールに従う）

## 結論

上記の通り抜け漏れなし。設計（SCR-002定義書の更新）に進む。
