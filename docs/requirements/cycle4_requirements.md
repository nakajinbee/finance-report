# サイクル4 要件定義

## プロダクト概要
サイクル1〜3でバックエンド（データ取得・精度）を中心に作り込んできたが、フロントエンドは
サイクル1の見た目のまま拡張を重ねており、画面間で共通のヘッダー・フッターがない、
グラフの見た目（配色・軸ラベル等）がページ・コンポーネントごとにバラバラ、という状態になっている。

## サイクル4の目標
1. **ヘッダー・フッターを作成**し、全画面で共通のナビゲーション・帰属表示を持たせる
2. **グラフを見やすくする**（配色の統一・軸ラベルの整理等）
3. 上記を実現するにあたり、**できるだけシンプルなデザイン・カラーリングで仮のデザインを作る**
   （本格的なデザインコンセプト・配色の決定はサイクル5・6で行う。今回はその前段階の
   「機能面の器」を作ることが目的であり、見た目の作り込みは意図的に最小限にとどめる）

---

## 機能要件

### FR-29：共通ヘッダー
- 全画面（SCR-001〜004）の上部に、共通のヘッダーを表示する
- 内容：アプリ名（仮称でよい）、「企業一覧」への導線
- 実装方法：`App.tsx`のルーティングに共通レイアウト（`Layout`コンポーネント）を導入し、
  各ページコンポーネントをその子要素として描画する
- ヘッダーの「企業一覧」導線は、クリックで`/companies`へ遷移する（DBが空の場合の挙動は
  既存のSCR-002の「データがありません」表示に任せる。ヘッダー側で分岐しない）

### FR-30：共通フッター
- 全画面の下部に、共通のフッターを表示する
- 内容：データ出典表示（「データ出典：EDINET（金融庁）」等）
- 過度な情報（バージョン番号、SNSリンク等）は今回のスコープに含めない（YAGNI）

### FR-31：グラフの見やすさ改善
- 現状、各グラフコンポーネント（`FinancialChart`・`CashFlowChart`・`RatioCategoryChart`）が
  それぞれ個別にTableau10系の色コードをハードコートしており、コンポーネントをまたいだ
  配色の一貫性がない。今回は色の値自体を1箇所（`lib/theme.ts`等）に集約し、各コンポーネントは
  そこから参照する形にする（配色そのものの再設計＝デザインコンセプト決定はサイクル5・6で行う。
  今回は「参照元を1つにまとめる」というリファクタリングが主眼）
- 各セクション（B/S・P/L・CF・財務分析指標の各カテゴリ）の見出し・余白などの
  視覚的な区切りを統一する（現状、`<h2>`・`<h3>`が混在し余白の付け方も揃っていない）

### FR-32：仮の配色・スタイル
- Tailwind CSS v4の`@theme`機能を使い、アプリ全体で使う基本トーン（背景色・文字色・
  アクセントカラー）を最小限定義する。「シンプルなデザイン」を優先し、装飾的な要素
  （グラデーション・アイコン画像等）は追加しない
- 本格的な配色・デザインコンセプトの検討はサイクル5・6のスコープであり、本サイクルの
  スタイルは「仮」であることを明示する（コード上のコメントで、サイクル5・6で置き換わる
  前提であることを記録する）

## 非機能要件

### NFR-09：影響範囲
本サイクルの変更はフロントエンドの見た目・共通レイアウトのみであり、バックエンド
（API・DB・EDINET連携）には一切変更を加えない。

### NFR-10：既存機能への影響
既存の全画面（SCR-001〜004）の機能（検索・ダウンロード・グラフ表示・指標トグル等）は
変更しない。ヘッダー・フッターの追加、配色の集約、見出し・余白の統一のみを行う。

### FR-33：仮デザインの「作り込み感」向上（2026-07-22追加）
- **経緯**：FR-29〜32で共通ヘッダー・フッター・水色アクセントを導入したが、まだ
  「最低限機能する仮の見た目」の域を出ていない（ページ幅が狭くMacBook標準の論理解像度に
  対して余白が余る、カード・影がなく要素の区切りが平坦、ボタン・空状態表示がページごとに
  バラバラ）。ユーザーから「本格的な配色決定（サイクル5・6）の前段階として、仮決めのままで
  いいので"作り込み感"を出したい」との要望があった
- **利用シーンの見直し**：MacBook Air/Pro 13〜14インチの標準論理解像度（1440px前後）を
  主戦場とし、下限1280px程度を想定する（PC専用、モバイル対応は引き続きスコープ外）。
  ウィンドウは可変であるため、固定`max-w`ではなく`w-full`＋`max-w-*`の流動的なレイアウトを
  維持しつつ、現状全体的に狭すぎる`max-w`の値を見直す
- **レイアウト幅**：データ量の多い画面（企業詳細・生データ確認）は`max-w-6xl`程度に拡大。
  フォーム的な画面（ダウンロード・企業一覧）は`max-w-2xl`程度に微増
- **レスポンシブデザイン**：PCでの利用を前提としつつ、ウィンドウ幅に応じてレイアウトが
  適応するようにする（固定幅の決め打ちにしない）。具体的には、画面幅が十分広い場合は
  カード（Panel）を**2列**に並べて表示し、狭い場合は1列に積む
  - 企業詳細画面：貸借対照表（B/S）・損益計算書（P/L）を2列に並べる。財務分析指標の
    4カテゴリ（収益性・効率性・安全性・投資指標）も2列×2行に並べる
  - 企業一覧画面：企業カードの一覧を2列に並べる
  - キャッシュフロー計算書は対になるセクションがないため、常に1列（幅いっぱい）のままとする
  - 2列になる幅の閾値はTailwindの`xl`（1280px）ブレークポイントを使う（想定する最小幅
    1280pxで2列表示に切り替わり、それより狭いウィンドウでは1列に積む）
- **カード化**：共通の`Panel`コンポーネントを新設し、各セクション・一覧行・テーブルを
  カード（角丸・枠線・薄い影）で区切る
- **ボタンの統一**：共通の`Button`コンポーネント（primary/secondary）を新設し、
  画面ごとにバラバラだったボタンのスタイルを統一する
- **空状態・ローディング表示の統一**：素のテキスト表示を、中央寄せ・余白を持つ表示に統一する
  （アニメーションは追加しない）
- **配色**：ユーザーが提示したCanvaの配色パレット「05. クールブルー」を採用する
  （後から変更可能という前提、2026-07-22ユーザー承認）

  | 名称 | 色コード | 用途 |
  |---|---|---|
  | deep aqua | `#003B46` | ヘッダー背景・強調テキスト |
  | ocean | `#07575B` | メインアクセント（ボタン・リンク・トグル選択状態） |
  | wave | `#66A5AD` | ホバー・枠線・サブアクセント |
  | seafoam | `#C4DFE6` | 薄い背景（フッター・カード内ハイライト） |

  グラフの系列色（`CHART_COLORS`、Tableau10ベース）は指標の見分けやすさを優先し変更しない
- **フォント**：システムフォント任せから、モダンなWebフォント（欧文：Inter、和文：Noto
  Sans JP）に変更する（ユーザー要望、2026-07-22。「流行りのものに」という抽象的な要望への
  対応として、モダンなSaaS製品で広く使われているInterを採用）
- **グラフの軸ラベル簡略化**：会計年度の軸ラベルを「2021年3月期」ではなく「2021」のように
  年だけの表示にする（ユーザー要望、2026-07-22）。`fiscal_year`フィールド自体
  （`date_format_policy.md`準拠、`period_end`から動的生成しない文字列）は変更せず、
  ツールチップ・表では引き続きフル表記（「2021年3月期」）を使う。軸ラベルの表示のみを
  簡略化する
