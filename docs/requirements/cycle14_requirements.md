# サイクル14 要件定義

## 背景

以下2点について、ユーザーから直接の修正依頼があった。

1. サイクル5でヘッダーを「白背景＋下部border＋テキストリンク」に変更したが、
   ユーザーから「ヘッダーの色がなくなった」との指摘があり、確認の結果
   「背景を水色系（ブランドカラー）で塗る」見た目に戻したいとのことだった
   （design_guideline.mdの該当箇所も合わせて更新する）。
2. 企業一覧画面（SCR-002）は、DBに保存済みの企業を常時全件カード表示し、
   検索バーで絞り込む設計だった。サイクル6以降、全上場企業（約3,800社）を
   一括登録するようになった結果、初期表示時に3,800件以上のカードが並ぶ
   状態になってしまっている。これを、検索または絞り込み条件を指定した
   ときだけ結果を表示する形に変更したい。あわせて、業種による絞り込みと
   ソート順の指定も追加する。

## 機能要件

### FR-61：ヘッダー背景色をブランドカラーに戻す

- `Header`の背景を`bg-brand`（ブランドカラー）に変更する
- ロゴ文字（「企業会計情報」）は`text-white`にする
- ナビリンク（「企業一覧」）は`text-brand-tint`、ホバー時`text-white`にする
- フッターの配色（`bg-gray-50`＋上部border）は変更しない
- design_guideline.mdの「ヘッダー・フッター」節を、この見た目に更新する

### FR-62：企業一覧画面（SCR-002）を「検索・絞り込み時のみ表示」に変更する

- 初期表示時（検索キーワード未入力・業種未選択）は、企業カード一覧を表示しない。
  かわりに「企業名・証券コードで検索するか、業種を選択してください」といった
  案内メッセージを表示する
- 検索キーワードが入力されている、または業種が選択されている場合のみ、
  その条件に一致する企業をカード一覧に表示する
- 検索・絞り込みの実行方式（APIを都度呼ばずフロントで絞り込む）は現状を維持する
  （`GET /api/companies`で取得済みの一覧に対してクライアント側でフィルタ・ソートする。
  API・DBの変更は不要）

### FR-63：業種による絞り込みを追加する

- 検索バーの近くに業種選択のセレクトボックスを追加する
- 選択肢は、取得済み企業一覧に含まれる業種（`sector`）の重複なしリスト
  （五十音順）＋「すべて」
- 「すべて」以外を選択した場合、その業種と一致する企業のみに絞り込む
  （キーワード検索と業種絞り込みは併用可能＝AND条件）
- `sector`が`null`（データ未取得）の企業は、業種選択の対象には含めない
  （選択肢に「業種不明」等のダミー項目を作らない）

### FR-64：ソート順の指定を追加する

- ソート順を選択するセレクトボックスを追加する
- 選択肢：証券コード昇順（デフォルト、現状維持）・証券コード降順・
  企業名昇順（あいうえお順）・企業名降順（2026-07-24追記：初期実装で企業名降順の
  選択肢が漏れていたため追加）
- 選択したソート順を、絞り込み後の企業カード一覧に適用する

## スコープ外

- バックエンド・API・DBの変更（クライアント側の絞り込み・ソートのみで完結するため）
- 業種のカテゴリ体系の見直し（EDINETの業種区分をそのまま使う）
- 検索条件のURL保存・ブラウザ履歴連携
