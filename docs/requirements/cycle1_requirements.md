# サイクル1 要件定義

## プロダクト概要
EDINET API を利用して、リクルートホールディングスの会計情報を5年分グラフ表示する WEB アプリケーション。

---

## サイクル1 の目標
**リクルートホールディングスの財務指標を5年分取得してグラフ表示する**
企業検索・カスタマイズ等の汎用機能はサイクル2以降に対応する。

### 対象企業
- 企業名：リクルートホールディングス
- 証券コード：6098
- 会計基準：IFRS（国際財務報告基準）
- 上場市場：東京証券取引所 プライム市場

---

## 機能要件

### FR-01：財務データ取得
- EDINET API からリクルートホールディングスの有価証券報告書（年次、`docTypeCode: "120"`）を直近5期分取得する
- 取得する財務指標（EDINET CSV の要素ID。実機検証済み、2026-07-19時点）：
  - 売上高（`jpcrp_cor:RevenueIFRSSummaryOfBusinessResults`）※経営指標等サマリーに5期分まとまって出力される
  - 営業利益（`jpigp_cor:OperatingProfitLossIFRS`）※連結損益計算書本体の値。書類（期）ごとに1件のみ
  - 純利益（`jpcrp_cor:ProfitLossAttributableToOwnersOfParentIFRSSummaryOfBusinessResults`）※親会社所有者帰属分。経営指標等サマリーに5期分まとまって出力される
  - 総資産（`jpcrp_cor:TotalAssetsIFRSSummaryOfBusinessResults`）※経営指標等サマリーに5期分まとまって出力される
  - 負債合計（`jpigp_cor:LiabilitiesIFRS`）※連結財政状態計算書本体の値。書類（期）ごとに1件のみ
- 各書類の対象会計期間（開始・終了日）を取得する

> ※ 当初は生の IFRS タクソノミ（`ifrs-full:*`）を想定していたが、EDINET が実際に払い出す CSV は
> EDINET 拡張タクソノミ（`jpcrp_cor:*` 経営指標等サマリー系、`jpigp_cor:*` 財務諸表本体系）を使用しているため、
> 実機検証（`memo/リクルートデータ取得メモ.md` 参照）の結果に合わせてタグ名を修正した。
> 売上高・純利益・総資産の3指標は最新1件の書類だけで5期分取得できるが、営業利益・負債合計の2指標は
> サマリーに含まれないため、5期分の書類（`docID`）をそれぞれ取得する必要がある。

### FR-02：グラフ表示
- 取得した5期分の指標を折れ線グラフまたは棒グラフで表示する
- 横軸：会計年度、縦軸：金額（単位：億円）
- 指標はタブまたはボタンで切り替えて表示できる（1画面に1指標）
- 各指標の直近値をグラフと合わせて数値でも表示する

### FR-03：エラー表示
- EDINET API エラー時は「データの取得に失敗しました」とメッセージ表示する
- 指標のデータが存在しない場合は「データなし」と表示する（アプリをクラッシュさせない）

---

## 非機能要件

### NFR-01：パフォーマンス
- 初期表示（データ取得〜グラフ描画）：15秒以内

### NFR-02：対応環境
- PC ブラウザ（Chrome 最新版）で動作すること

### NFR-03：セキュリティ
- EDINET API キーはサーバーサイドの環境変数で管理し、フロントエンドに露出させない

---

## スコープ外（サイクル2以降）
- 企業名・証券コードによる企業検索
- 複数企業の切り替え・比較
- 表示指標のカスタマイズ
- J-GAAP / US GAAP 対応
- 複数年の動的な期間選択
- モバイル対応
- **負債合計の内訳化（流動負債・固定負債）**：現状「負債合計」のみを1指標として保持しているが、
  流動負債と固定負債を分けて保存・表示したいという要望あり。EDINET CSVには
  `jpigp_cor:TotalCurrentLiabilitiesIFRS`（流動負債）が個別タグとして存在することを実機確認済み。
  → この個別ニーズは、サイクル2で計画中の「EDINET報告データの網羅的なDB保存」
  （[cycle2_requirements.md](cycle2_requirements.md) FR-04/FR-05）に含まれる形で解決する想定

---

## 前提・制約
- データソースは EDINET API（無料・要APIキー）
- リクルートホールディングスは IFRS 採用のため、IFRS タクソノミのタグを使用する
- XBRL のパースにはサーバーサイドのライブラリを使用する
- 5期分のデータが EDINET に存在することを前提とする
