# ユースケース一覧

[product/concept.md](concept.md)の提供価値をもとに、大項目・中項目・小項目の
3階層で整理した。以降の機能追加は、ここに挙げた中項目（または規模に応じて
小項目）を単位に、画面フロー→画面→API→バッチの順で設計・実装していく
（[cycle-workflow](../../.claude/skills/cycle-workflow/SKILL.md)参照）。

このファイルは一覧のみを持つ、常に最終断面の軽量なインデックス。各中項目の
詳細（画面遷移・関連機能・依存関係等）は
`docs/product/use_case_details/UC-<大項目番号>_<大項目名>/UC-<大項目番号>-<中項目番号>_<中項目名>.md`
に記載する（詳細ファイルが作成済みの中項目のみ、下記一覧からリンクする）。

## ID体系

各ユースケースに`UC-<大項目番号>`／`UC-<大項目番号>-<中項目番号>`／
`UC-<大項目番号>-<中項目番号>-<小項目番号>`のIDを振る（例：大項目1の中項目1は
`UC-1-1`、その1つ目の小項目は`UC-1-1-1`）。以降の要件定義・設計書からこのIDで
参照する。

各中項目の「状態」は既存の画面・APIと突き合わせて判定したもの（実現済み／
部分実現／未実現）。実装が進むにつれて更新する。

## UC-1：投資判断のための企業調査

### [UC-1-1 特定企業の深掘り調査](use_case_details/UC-1_投資判断のための企業調査/UC-1-1_特定企業の深掘り調査.md)（状態：実現済み）

- UC-1-1-2 財務分析指標を確認する
- UC-1-1-3 事業内容・リスク情報等の定性情報を確認する

### UC-1-2 複数企業の比較（状態：未実現）
- UC-1-2-1 同業他社と指標を横並び比較する
- UC-1-2-2 指標ランキングで相対位置を把握する

### UC-1-3 業界・セクター全体の俯瞰（状態：未実現）
- UC-1-3-1 売上高ベースで業界内のシェア（相対位置）を見る
- UC-1-3-2 時価総額・利益の企業ランキングを見る（全体、または業界で絞り込み可能）

### UC-1-4 保有銘柄・注目企業の定点観測（状態：未実現）
- UC-1-4-1 ウォッチリストに登録する
- UC-1-4-2 新しい決算・開示があったことを検知する

### UC-1-5 株価変動の要因調査（状態：未実現）
- UC-1-5-1 株価推移を確認する
- UC-1-5-2 変動タイミングと変動の原因を、開示書類・時事ニュース・他の関連指標を
  もとに分析する
- **依存関係**：株価データを取得・保存する仕組みが現状存在しない
  （[IDEA-12](../ideas/IDEA-12_stock_price_check.md)が関連するが未着手）。着手する
  場合はまずこのデータ取得基盤が前提になる

## UC-3：データ蓄積・時系列分析

### UC-3-1 データを定期的に収集・蓄積する（状態：部分実現）
- UC-3-1-1 特定のスパン（日次等）でデータを収集する
- UC-3-1-2 蓄積したデータをもとに素早く表示・分析できるようにする

### UC-3-2 同一企業の指標の時系列追跡（状態：部分実現）
- UC-3-2-1 複数年度の指標推移をグラフで見る
- UC-3-2-2 成長性・収益性の変化を確認する

### UC-3-3 定点観測企業群の変化の振り返り（状態：未実現）
- UC-3-3-1 ウォッチ対象企業をまとめて振り返る
- UC-3-3-2 一定期間ごとの変化サマリーを見る
- **依存関係**：UC-1-4（保有銘柄・注目企業の定点観測）が前提。ウォッチリストの
  概念自体がまだ存在しないため、UC-1-4より先に着手することはできない

### UC-3-4 株価と決算内容の対応関係の蓄積（状態：未実現）
- UC-3-4-1 株価変動タイミングと決算内容を紐づけて記録する
- UC-3-4-2 蓄積したデータから傾向を分析する
- **依存関係**：UC-1-5と同じく株価データ取得基盤が前提（未着手）

### UC-3-5 独自の理論株価算出・割安株スクリーニング（状態：未実現）
- UC-3-5-1 蓄積データをもとに独自の理論株価算出モデルを構築する
- UC-3-5-2 理論値より株価が割安な銘柄を探す
- **依存関係**：株価データ取得基盤（UC-1-5・UC-3-4）に加え、UC-3-1〜3-4での
  蓄積データの蓄積量にも依存する。他のユースケースより着手時期は後になる見込み
