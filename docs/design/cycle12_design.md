## 1. `docs/product/use_cases.md`の構成

`docs/product/concept.md`と同じ位置づけ（常に最終断面のみを書く）。構成：

```markdown
# ユースケース一覧

## 大項目1：投資判断のための企業調査
### 1-1 〜
- 小項目
- 小項目
（状態：実現済み／部分実現／未実現）
```

各中項目に「状態」（実現済み／部分実現／未実現）を注記する。既存の
`screen_list.md`・`api_list.md`を`grep`・読解で確認し、以下の通り判定した：

| 中項目 | 状態 | 根拠 |
|---|---|---|
| 1-1 特定企業の深掘り調査 | 部分実現 | 財務諸表(SCR-003)・財務分析指標(API-COM-005)は実現済み。定性情報（事業内容・リスク）は未実現 |
| 1-2 複数企業の比較 | 未実現 | 対応する画面・APIなし（[IDEA-05](../ideas/IDEA-05_company_comparison_page.md)が該当） |
| 1-3 業界・セクター全体の俯瞰 | 未実現 | 対応する画面・APIなし（[IDEA-04](../ideas/IDEA-04_company_list_sector_filter.md)・[IDEA-06](../ideas/IDEA-06_sector_share_map.md)が関連） |
| 1-4 保有銘柄・注目企業の定点観測 | 未実現 | ウォッチリスト・更新検知の概念自体が存在しない |
| 1-5 株価変動の要因調査 | 未実現 | 株価データを一切保持していない（[IDEA-12](../ideas/IDEA-12_stock_price_check.md)が関連） |
| 2-1 実データで会計の数値を理解する | 部分実現 | 財務諸表・指標の閲覧はできるが、「規模感」の解説はない |
| 2-2 会計基準・指標の意味を都度調べる | 未実現 | 指標の定義・用語解説のUIが存在しない |
| 2-3 学習モードと通常モードを切り替える | 未実現 | モード切り替えの概念自体が存在しない |
| 3-1 データを定期的に収集・蓄積する | 部分実現 | BATCH-003（書類一覧）・BATCH-004（書類本体）は実装済みだが、全社規模の本実行・日次自動化は未完（サイクル9スコープ外） |
| 3-2 同一企業の指標の時系列追跡 | 部分実現 | SCR-003で年度範囲指定のグラフ表示はできるが、「保有銘柄群をまとめて」の視点はない |
| 3-3 定点観測企業群の変化の振り返り | 未実現 | 1-4（定点観測）が未実現のため前提から成立していない |
| 3-4 株価と決算内容の対応関係の蓄積 | 未実現 | 株価データ自体が存在しない |
| 3-5 独自の理論株価算出・割安株スクリーニング | 未実現 | 技術的難度が高く、他のユースケースの実装状況に依存する |
