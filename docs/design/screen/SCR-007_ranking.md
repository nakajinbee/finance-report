# SCR-007 ランキング画面

> 画面項目とAPIレスポンスの対応は [items/SCR-007_items.md](items/SCR-007_items.md) を参照。

## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-007 |
| 画面名 | ランキング画面 |
| パス | `/ranking` |
| 呼び出し元 | 共通ヘッダーの「ランキング」リンク、SCR-003の「この企業の業種内での順位を見る」リンク |
| 関連サイクル | サイクル15 FR-67・FR-68（UC-1-2-2：指標ランキングで相対位置を把握する） |

SCR-003から遷移した場合、その企業の`sector`を初期値として業種絞り込みに
引き継ぐ（指標は未選択のまま、ユーザーが選ぶ）。

**2026-07-24訂正**：当初は指標・業種の選択が変わるたびに即座にAPI-COM-007を
呼び出す設計だったが、ユーザーから「指標を選んだ瞬間に業種すべてのデータが
表示されてしまう」との指摘があり、「表示」ボタンを押したときだけデータを
取得・表示する方式に変更した（指標・業種の選択自体は即座に反映されるプレビュー
状態で、実際のデータ取得は明示的な操作を必要とする）。

## 機能定義

### 1. 指標選択

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 選択肢 | API-COM-007が受け付ける`metric`の一覧（FR-67参照。財務諸表7項目＋財務分析
  指標12項目、日本語ラベルで表示） |
| 初期状態 | 未選択 |

### 2. 業種絞り込み

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 選択肢 | SCR-002と同じ生成方法（取得済み企業一覧から`sector`の重複除去、五十音順、
  `null`除外）＋「すべて」（先頭） |
| 初期状態 | 「すべて」。ただしSCR-003から遷移した場合はその企業の`sector`を初期選択する |

### 3. 昇順・降順切り替え

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 選択肢 | 降順（初期状態）・昇順 |
| 適用方法 | API-COM-007から取得済みの結果をフロントで並び替える（API再呼び出しなし。
  「表示」ボタンを押していなくても、選択自体は変更できる） |

### 4. 表示ボタン（2026-07-24追加）

| 項目 | 内容 |
|------|------|
| 表示位置 | 昇順・降順切り替えの隣 |
| 活性化条件 | 指標が選択されている場合のみ活性化 |
| 押下時の処理 | その時点で選択中の指標・業種でAPI-COM-007を呼び出し、ランキング表を
  表示する。指標・業種セレクトを変更しただけでは呼び出さない
  （変更前の表示内容がそのまま残る） |

### 5. ランキング表

| 項目 | 内容 |
|------|------|
| 表示条件 | 「表示」ボタンを押した後、指標が選択されている場合のみ |
| データ取得元 | `GET /api/companies/ranking?metric=&sector=`（業種「すべて」の場合は
  `sector`パラメータなしで呼び出す） |
| 表示列 | 順位（フロントで採番）・企業名・証券コード・業種・選択指標の値 |
| クリック時の処理 | 行クリックでSCR-003（企業詳細画面）へ遷移 |
| 結果0件の場合 | 「該当するデータがありません」 |

### 6. 案内メッセージ

| 項目 | 内容 |
|------|------|
| 表示条件 | 「表示」ボタンをまだ押していない場合 |
| 表示内容 | 「指標・業種を選んで「表示」を押してください」 |

## 画面状態

| 状態 | 説明 | 表示内容 |
|------|------|----------|
| 未表示 | 「表示」ボタンをまだ押していない（初期表示・指標未選択を含む） | 案内メッセージのみ、表は非表示 |
| ローディング | API-COM-007呼び出し中 | 「データを読み込んでいます...」 |
| 正常 | 「表示」ボタン押下後、結果1件以上 | ランキング表 |
| 結果0件 | 「表示」ボタン押下後、該当データなし | 「該当するデータがありません」 |
| エラー | API-COM-007がエラーを返却 | 「データの取得に失敗しました。しばらくしてから再度お試しください。」 |
