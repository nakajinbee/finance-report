# SCR-001 ダウンロード画面

> 画面項目とAPIレスポンスの対応は [items/SCR-001_items.md](items/SCR-001_items.md) を参照。

## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-001 |
| 画面名 | ダウンロード画面 |
| パス | `/download` |
| 呼び出し元 | SCR-002（企業一覧画面）の「データを追加取得する」ボタン |
| 初期遷移 | アプリ起動時に DB が空の場合、自動的にこの画面へリダイレクトする |

サイクル2で企業検索（FR-07）・取得期間指定（FR-09）に対応した。
サイクル1では「対象企業：リクルートホールディングス固定」「取得期間：直近5期分固定」だったが、
サイクル2では両方をユーザーが指定する。

## 機能定義

### 1. 企業検索ボックス

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 検索対象 | EDINETに登録された全上場企業（DBに未保存の企業も含む） |
| 検索タイミング | 入力のたびに`GET /api/edinet/companies/search?q=`を呼び出す（デバウンス要、間隔は実装時に決める） |
| 使用 API | `GET /api/edinet/companies/search` |

### 2. 検索結果一覧・選択

| 項目 | 内容 |
|------|------|
| 表示条件 | 検索結果が1件以上ある場合 |
| クリック時の処理 | その企業を「選択中」として確定し、検索結果一覧を閉じる |
| 選択中の表示 | 「選択中：{企業名}」を検索ボックスの下に表示 |

### 3. 取得期間の指定

| 項目 | 内容 |
|------|------|
| 選択肢 | 「全期間」／「期間指定（開始年〜終了年）」のラジオボタン |
| 初期選択 | 「全期間」 |
| 期間指定時の入力 | 開始年・終了年をそれぞれ数値入力（またはセレクタ） |
| 全期間選択時の注意書き | 「EDINETの制約により、実際に取得できるのは最大10年分です」を表示（FR-09） |

### 4. データ取得ボタン

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 活性条件 | 企業が選択されている場合のみ活性（未選択時はグレーアウト） |
| 押下時の処理 | `POST /api/download`（`company_code`・`edinet_code`・`period`を送信）を呼び出してダウンロードを開始する |
| 押下後の状態 | ボタンを非活性化し、取得ログエリアを表示する |
| 再活性化タイミング | 選択中の企業のダウンロードが完了（成功・エラー問わず）した時点で再活性化する |
| 使用 API | `POST /api/download` |

### 5. 取得ログエリア

| 項目 | 内容 |
|------|------|
| 表示条件 | 取得ボタン押下後（完了・エラー後も表示し続ける） |
| 更新方法 | `GET /api/download/status?company_code=`（選択中の企業）を1秒ごとにポーリングし、全件完了または全件エラー確定時にポーリングを停止する |
| 表示内容 | 各期の取得状況をリスト表示。件数は取得期間の指定により可変（サイクル1は5件固定だったが、サイクル2は「全期間」「期間指定」の内容に応じて変わる） |
| アイコン定義 | ✓＝取得完了、⏳＝取得中、→＝スキップ（既存データがあるためEDINETへ再アクセスしなかった、FR-11）、✗＝エラー |
| 使用 API | `GET /api/download/status` |

### 6. 企業一覧へボタン

| 項目 | 内容 |
|------|------|
| 表示条件 | 選択中企業のダウンロードが完了し、かつ1件以上成功した場合のみ表示 |
| 押下時の処理 | SCR-002（企業一覧画面）へ遷移 |

## 画面状態

| 状態 | 説明 | 表示内容 |
|------|------|----------|
| 初期 | 企業未選択 | 検索ボックスのみ活性。取得ボタンは非活性 |
| 企業選択済み | 企業選択後、ボタン押下前 | 取得ボタン活性。ログエリアは非表示 |
| 取得中 | ダウンロード進行中 | ボタン非活性（グレーアウト）＋ログリスト表示 |
| 完了（全件成功） | 指定期間分すべて取得完了（`skipped`も成功扱い） | ボタン再活性＋ログリスト＋「企業一覧へ」ボタン表示 |
| 完了（一部エラー） | 一部の期でエラーが発生 | ボタン再活性＋ログリスト（✗のある行を含む）＋「企業一覧へ」ボタン表示 |
| 完了（全件エラー） | 全期でエラーが発生 | ボタン再活性＋ログリスト＋「データの取得に失敗しました。再度お試しください。」を表示。「企業一覧へ」ボタンは非表示 |

`skipped`（FR-11、既存データがあるため再アクセスをスキップ）は`done`と同じく「成功」として扱う。
「企業一覧へ」ボタンの表示条件（`logs[]`に1件以上`status == "done"`を含む）も、`skipped`を
含む場合に拡張する（`docs/design/screen/items/SCR-001_items.md` SCR-001-10参照）。
