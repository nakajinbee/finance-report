# SCR-001 ダウンロード画面

> 画面項目とAPIレスポンスの対応は [items/SCR-001_items.md](items/SCR-001_items.md) を参照。

## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-001 |
| 画面名 | ダウンロード画面 |
| パス | `/download` |
| 呼び出し元 | SCR-002（企業一覧画面）の「データを追加取得する」ボタン |
| 初期遷移 | アプリ起動時に DB が空の場合、自動的にこの画面へリダイレクトする |

## 機能定義

### 1. データ取得ボタン

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 押下時の処理 | `POST /api/download` を呼び出してダウンロードを開始する |
| 押下後の状態 | ボタンを非活性化（グレーアウト）し、取得ログエリアを表示する |
| 再活性化タイミング | 全期のダウンロードが完了（成功・エラー問わず）した時点でボタンを再活性化する |
| 使用 API | `POST /api/download` |

### 2. 取得ログエリア

| 項目 | 内容 |
|------|------|
| 表示条件 | 取得ボタン押下後（完了・エラー後も表示し続ける） |
| 更新方法 | `GET /api/download/status` を1秒ごとにポーリングし、全件完了または全件エラー確定時にポーリングを停止する |
| 表示内容 | 各期の取得状況をリスト表示。各行に「{アイコン} {会計年度} {ステータスメッセージ}」を表示 |
| アイコン定義 | ✓＝取得完了、⏳＝取得中、✗＝エラー |
| 使用 API | `GET /api/download/status` |

### 3. 企業一覧へボタン

| 項目 | 内容 |
|------|------|
| 表示条件 | 全期のダウンロードが完了し、かつ1件以上成功した場合のみ表示 |
| 押下時の処理 | SCR-002（企業一覧画面）へ遷移 |

## 画面状態

| 状態 | 説明 | 表示内容 |
|------|------|----------|
| 初期 | ボタン押下前 | 取得ボタンのみ表示。ログエリアは非表示 |
| 取得中 | ダウンロード進行中 | ボタン非活性（グレーアウト）＋ログリスト表示 |
| 完了（全件成功） | 全期分取得完了 | ボタン再活性＋ログリスト＋「企業一覧へ」ボタン表示 |
| 完了（一部エラー） | 一部の期でエラーが発生 | ボタン再活性＋ログリスト（✗のある行を含む）＋「企業一覧へ」ボタン表示 |
| 完了（全件エラー） | 全期でエラーが発生 | ボタン再活性＋ログリスト＋「データの取得に失敗しました。再度お試しください。」を表示。「企業一覧へ」ボタンは非表示 |
