# SCR-004 保存済みデータ確認画面

> 画面項目とAPIレスポンスの対応は [items/SCR-004_items.md](items/SCR-004_items.md) を参照。

## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-004 |
| 画面名 | 保存済みデータ確認画面 |
| パス | `/companies/:code/facts` |
| 呼び出し元 | SCR-003（企業詳細画面）の「生データを確認」リンク |

サイクル2新規（`docs/requirements/cycle2_requirements.md` FR-16）。TBL-003 factsに実際に
どんなデータが保存されているかを確認するための画面。**グラフ化・集計・分析機能は持たない**、
保存済みの生データをそのまま一覧表示するだけの確認・デバッグ用途の画面である
（「データ分析画面・レポート機能」（スコープ外、サイクル3以降）とは明確に区別する）。

## 機能定義

### 1. 戻るリンク

| 項目 | 内容 |
|------|------|
| 表示条件 | 常時表示 |
| 押下時の処理 | SCR-003（企業詳細画面）へ遷移 |

### 2. 要素IDでの絞り込み

| 項目 | 内容 |
|------|------|
| 入力形式 | テキスト入力（部分一致） |
| 検索タイミング | 入力のたびに`GET /api/companies/{code}/facts?element_id=`を再呼び出しする
  （件数が多くフロント内絞り込みには不向きなため、サーバー側で絞り込む） |

### 3. 期間での絞り込み

| 項目 | 内容 |
|------|------|
| 選択形式 | セレクタ（「すべて」＋DBに保存済みの`period_end`一覧） |
| 選択時の処理 | `GET /api/companies/{code}/facts?period_end=`を再呼び出しする |

### 4. ファクト一覧テーブル

| 項目 | 内容 |
|------|------|
| 表示条件 | 検索結果が1件以上ある場合 |
| 表示列 | 要素ID／項目名／書類種別（年次・半期）／期間／連結個別／値／単位 |
| ソート順 | 期間降順、同一期間内は要素ID昇順（実装時に調整可） |
| 使用 API | `GET /api/companies/{code}/facts` |
| ページング | 今回は行わない（`cycle2_requirements.md` FR-16参照。データ量が実際に問題になったら検討） |

## 画面状態

| 状態 | 説明 | 表示内容 |
|------|------|----------|
| ローディング | API 呼び出し中 | スピナー＋「読み込み中...」 |
| 正常（データあり） | ファクトが1件以上 | 絞り込みUI＋一覧テーブル |
| 絞り込み結果0件 | 絞り込み条件に一致するデータがない | 「条件に一致するデータが見つかりませんでした」 |
| データなし | その企業のファクトが0件（ダウンロード未実施等） | 「データがありません」 |
| エラー | API エラー | 「データの取得に失敗しました。しばらくしてから再度お試しください。」 |

| API | 用途 |
|-----|------|
| `GET /api/companies/{code}/facts` | 保存済みファクトの一覧取得（絞り込み条件付き） |
