# SCR-004 保存済みデータ確認画面 画面項目定義

親画面：[../SCR-004_facts_browser.md](../SCR-004_facts_browser.md)

各画面項目が、どのAPI（[api_list.md](../../api/api_list.md)参照）の、どのレスポンス項目から来ているかを定義する。
サイクル2新規（FR-16）。

---

## 項目一覧

| 項目ID | 画面項目名 | 表示位置 | 参照元API | 参照元レスポンス項目 | 表示形式・整形ルール | 備考 |
|---|---|---|---|---|---|---|
| SCR-004-01 | 戻るリンク | 画面左上 | なし（画面遷移のみ） | ― | 押下でSCR-003へ遷移。APIは呼ばない | |
| SCR-004-02 | 企業名見出し | 画面上部 | API-COM-001<br>`GET /api/companies`（SCR-003から遷移時に既に取得済みの企業名を引き継ぐ想定） | `Company.name` | 「{企業名} の保存済みデータ」 | |
| SCR-004-03 | 要素IDでの絞り込み | 見出し下 | API-COM-004<br>`GET /api/companies/{code}/facts?element_id=` | リクエストの`element_id`パラメータに入力値を渡す | 入力のたびにAPIを再呼び出し（部分一致） | |
| SCR-004-04 | 期間での絞り込み | 絞り込み欄 | API-COM-004<br>`?period_end=` | レスポンス`FactRecord[].period_end`の一覧から選択肢を生成 | セレクタ。「すべて」＋期間一覧 | |
| SCR-004-05 | 件数表示 | 絞り込み欄の下 | API-COM-004 | レスポンス配列の要素数 | 「{件数}件表示」 | |
| SCR-004-06 | ファクト一覧テーブル：要素ID列 | テーブル | API-COM-004 | `FactRecord.element_id` | そのまま表示 | |
| SCR-004-07 | ファクト一覧テーブル：項目名列 | テーブル | API-COM-004 | `FactRecord.element_name` | `null`の場合は空欄 | |
| SCR-004-08 | ファクト一覧テーブル：書類種別列 | テーブル | API-COM-004 | `FactRecord.doc_type_code` | `120`→「年次」、`160`→「半期」に変換 | |
| SCR-004-09 | ファクト一覧テーブル：期間列 | テーブル | API-COM-004 | `FactRecord.period_end` | そのまま表示（YYYY-MM-DD） | |
| SCR-004-10 | ファクト一覧テーブル：値・単位列 | テーブル | API-COM-004 | `FactRecord.value` / `FactRecord.unit` | 桁区切りのみ整形（億円/兆円変換はしない。生データ確認用のため） | |

## 画面状態とAPIレスポンスの対応

| 画面状態 | 判定に使うレスポンス項目 |
|---|---|
| ローディング | API-COM-004 呼び出し中（応答待ち） |
| 正常（データあり） | API-COM-004 のレスポンス配列が1件以上 |
| 絞り込み結果0件 | 絞り込み条件付きAPI呼び出しの結果が0件 |
| データなし | 絞り込みなしでもAPI-COM-004 の結果が0件（ダウンロード未実施等） |
| エラー | API-COM-004 が `404`（企業なし）または `500` エラーを返却 |
