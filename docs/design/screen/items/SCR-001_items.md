# SCR-001 ダウンロード画面 画面項目定義

親画面：[../SCR-001_download.md](../SCR-001_download.md)

各画面項目が、どのAPI（[api_list.md](../../api/api_list.md)参照）の、どのレスポンス項目から来ているかを定義する。

---

## 項目一覧

| 項目ID | 画面項目名 | 表示位置 | 参照元API | 参照元レスポンス項目 | 表示形式・整形ルール | 備考 |
|---|---|---|---|---|---|---|
| SCR-001-01 | 対象企業名 | 画面上部「対象企業：」 | なし（固定値） | ― | 「リクルートホールディングス」を固定表示 | サイクル1は単一企業固定。API化はサイクル2（企業検索対応）以降 |
| SCR-001-02 | 取得期間 | 画面上部「取得期間：」 | なし（固定値） | ― | 「直近5期分」を固定表示 | 同上 |
| SCR-001-03 | データ取得ボタン | 中央ボタン | API-EDN-001<br>`POST /api/download` | レスポンス `status`（`started`固定） | 押下時にAPIを呼び、`202`受信でボタンを非活性化しログエリア表示に遷移 | `409`（多重実行）時はボタンを非活性のまま維持しエラー表示 |
| SCR-001-04 | 取得ログ：各行のアイコン | ログリスト各行の先頭 | API-EDN-002<br>`GET /api/download/status` | `logs[].status` | `done`→✓／`in_progress`→⏳／`error`→✗ に変換して表示 | 1秒間隔でポーリングして更新 |
| SCR-001-05 | 取得ログ：会計年度 | ログリスト各行 | API-EDN-002 | `logs[].fiscal_year` | そのまま文字列表示（例：「2023年3月期」） | |
| SCR-001-06 | 取得ログ：ステータスメッセージ | ログリスト各行 | API-EDN-002 | `logs[].message` | そのまま文字列表示（例：「取得完了」） | |
| SCR-001-07 | 全体進捗判定（ポーリング停止・画面状態切替） | （非表示・内部制御） | API-EDN-002 | トップレベル `status`（`idle`/`in_progress`/`done`/`error`） | `done`または`error`確定でポーリング停止しボタン再活性化 | `logs[]`全件の集計ではなくトップレベル`status`を正とする |
| SCR-001-08 | 企業一覧へボタン | ログエリア下部 | API-EDN-002 | `status == "done"` かつ `logs[]` に1件以上 `status == "done"` を含む | 条件を満たす場合のみ描画。押下でSCR-002へ遷移（API呼び出しなし） | 全件エラー時（`status == "error"`）は非表示 |

---

## 画面状態とAPIレスポンスの対応

| 画面状態 | 判定に使うレスポンス項目 |
|---|---|
| 初期 | ボタン未押下（APIコールなし） |
| 取得中 | API-EDN-002 `status == "in_progress"` |
| 完了（全件成功） | API-EDN-002 `status == "done"` かつ `logs[]` 全件 `status == "done"` |
| 完了（一部エラー） | API-EDN-002 `status == "done"` かつ `logs[]` に `status == "error"` を含む |
| 完了（全件エラー） | API-EDN-002 `status == "error"` |
