# SCR-007 ランキング画面 画面項目定義

親画面：[../SCR-007_ranking.md](../SCR-007_ranking.md)

各画面項目が、どのAPI（[api_list.md](../../api/api_list.md)参照）の、どのレスポンス項目から来ているかを定義する。

---

## 項目一覧

| 項目ID | 画面項目名 | 表示位置 | 参照元API | 参照元レスポンス項目 | 表示形式・整形ルール | 備考 |
|---|---|---|---|---|---|---|
| SCR-007-01 | 指標選択セレクト | 画面上部 | なし（フロント内処理） | ― | API-COM-007の`metric`一覧を日本語ラベルで表示 | 初期状態は未選択 |
| SCR-007-02 | 業種絞り込みセレクト | 指標選択の隣 | API-COM-001 | `Company.sector`（重複除去） | SCR-002と同じ生成方法。「すべて」＋五十音順一覧 | SCR-003から遷移時は当該企業の`sector`を初期選択 |
| SCR-007-03 | 昇順・降順セレクト | 業種絞り込みの隣 | なし（フロント内処理） | ― | 「降順」（初期値）・「昇順」 | 表示済みの結果をフロントで並び替え。API呼び出しなし |
| SCR-007-04 | 表示ボタン（2026-07-24追加） | 昇順・降順セレクトの隣 | なし（画面遷移・API呼び出しの起点） | ― | 押下時点の指標・業種でAPI-COM-007を呼び出す | 指標未選択時は無効化。選択を変えただけでは呼び出さない |
| SCR-007-05 | 案内メッセージ | 表の位置 | なし（フロント内処理） | ― | 「指標・業種を選んで「表示」を押してください」 | 表示ボタンをまだ押していない場合のみ表示 |
| SCR-007-06 | ランキング表（コンテナ） | 画面中央 | API-COM-007 | レスポンス配列 | 昇順・降順セレクトの選択に従いソートして描画 | 表示ボタン押下後のみ表示 |
| SCR-007-07 | ランキング表：順位 | 各行1列目 | なし（フロント内処理） | ― | ソート後の配列インデックス+1 | |
| SCR-007-08 | ランキング表：企業名・証券コード・業種 | 各行2〜4列目 | API-COM-007 | `name`・`code`・`sector` | そのまま表示 | |
| SCR-007-09 | ランキング表：値 | 各行5列目 | API-COM-007 | `value` | 選択中の指標に応じてSCR-003と同じ整形ルール（金額／%／回転率等） | |
| SCR-007-10 | ランキング表：行クリック時の遷移先 | 各行 | API-COM-007 | `code` | `/companies/{code}`へ遷移 | |
| SCR-007-11 | 結果0件メッセージ | 表の位置 | API-COM-007 | レスポンス配列が空 | 「該当するデータがありません」 | |

## 画面状態とAPIレスポンスの対応

| 画面状態 | 判定に使うレスポンス項目 |
|---|---|
| 未表示 | 表示ボタンをまだ押していない（指標未選択を含む） |
| ローディング | API-COM-007 呼び出し中 |
| 正常 | API-COM-007のレスポンス配列が1件以上 |
| 結果0件 | API-COM-007のレスポンス配列が0件 |
| エラー | API-COM-007 が `500` エラーを返却（`Error`スキーマ） |
