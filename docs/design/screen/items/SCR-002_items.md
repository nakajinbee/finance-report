# SCR-002 企業一覧画面 画面項目定義

親画面：[../SCR-002_company_list.md](../SCR-002_company_list.md)

各画面項目が、どのAPI（[api_list.md](../../api/api_list.md)参照）の、どのレスポンス項目から来ているかを定義する。

---

## 項目一覧

| 項目ID | 画面項目名 | 表示位置 | 参照元API | 参照元レスポンス項目 | 表示形式・整形ルール | 備考 |
|---|---|---|---|---|---|---|
| SCR-002-01 | 検索バー | 画面上部 | なし（フロント内処理） | ― | API-COM-001で取得済みの一覧をフロントで絞り込む。API再呼び出しなし | 検索対象はSCR-002-04の企業名・証券コード |
| SCR-002-02 | 件数表示「{全件数}件中{表示件数}件表示」 | 検索バー下 | API-COM-001<br>`GET /api/companies` | 全件数＝レスポンス配列（`Company[]`）の要素数、表示件数＝検索バー絞り込み後の件数 | 「3件中3件表示」の形式で表示 | 表示件数はフロント側の絞り込み結果を都度カウント |
| SCR-002-03 | 企業カード一覧（コンテナ） | 画面中央 | API-COM-001 | レスポンス配列 `Company[]` | 証券コード昇順にソートして描画 | 初回のみAPI呼び出し。以降の検索絞り込みはフロントで処理 |
| SCR-002-04 | 企業カード：企業名 | カード内1行目 | API-COM-001 | `Company.name` | そのまま表示（例：「リクルートホールディングス」） | |
| SCR-002-05 | 企業カード：証券コード | カード内2行目 | API-COM-001 | `Company.code` | そのまま表示（例：「6098」） | |
| SCR-002-06 | 企業カード：業種 | カード内2行目 | API-COM-001 | `Company.sector` | そのまま表示（例：「サービス業」）。`null`の場合は非表示 | `Company.sector`は`nullable: true` |
| SCR-002-07 | 企業カード：会計基準 | カード内2行目 | API-COM-001 | `Company.accounting_standard` | そのまま表示（例：「IFRS」） | |
| SCR-002-08 | 企業カード：クリック時の遷移先 | カードクリック | API-COM-001 | `Company.code` | `/companies/{code}` へ遷移し、SCR-003で`code`をパスパラメータとして使用 | SCR-003側は`API-COM-002`の`code`パラメータに引き継ぐ |
| SCR-002-09 | データ追加取得ボタン | 画面下部 | なし（画面遷移のみ） | ― | 押下でSCR-001へ遷移。APIは呼ばない | |

---

## 画面状態とAPIレスポンスの対応

| 画面状態 | 判定に使うレスポンス項目 |
|---|---|
| ローディング | API-COM-001 呼び出し中（応答待ち） |
| 正常（データあり） | API-COM-001 のレスポンス配列が1件以上 |
| 検索結果0件 | API-COM-001 取得後、フロントの検索絞り込み結果が0件 |
| 正常（データなし） | API-COM-001 のレスポンス配列が0件 |
| エラー | API-COM-001 が `500` エラーを返却（`Error`スキーマ） |
