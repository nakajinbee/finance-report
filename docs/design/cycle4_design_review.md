## 追記（2026-07-22）：FR-33のレビュー

- [x] 配色変更が`CHART_COLORS`等のグラフ系列色に影響しないか　→ 4.1節で明記済み
  （グラフ系列色は変更しない）
- [x] `Panel`・`Button`の責務が単純明快か　→ どちらも見た目のみのラッパーで、
  ロジックを持たない。過度な抽象化（バリアントの増やしすぎ等）を避け、
  `Button`は`primary`/`secondary`の2種類に留めた
- [x] 「戻る」リンクを`Button`化しない判断の妥当性　→ ボタンではなくナビゲーションの
  意味合いが強いため、既存の見た目のまま残す判断は妥当
- [x] 空状態表示をコンポーネント化しない判断　→ パターンが単純（`div`1つ＋クラス）なため、
  コンポーネント化はオーバーエンジニアリングと判断。既存のプロジェクト方針
  （過度な抽象化を避ける）と整合

抜け漏れ・矛盾は見つからなかった。

## 判定：実装フェーズへ進んで問題なし

抜け漏れ・矛盾は見つからなかった。実装は`lib/theme.ts`（FR-32）→
`Header`/`Footer`/`Layout`（FR-29〜30）→ 各グラフコンポーネントの色参照置き換え・
見出し統一（FR-31）の順で進める。FR-33は配色更新→`Panel`/`Button`新設→
各画面への適用（幅・カード化・2列グリッド・空状態表示）の順で進める。
